AMI Web Framework (AWF)

Documentation

The ATLAS Metadata Interface (AMI) is a very generic tool ecosystem for metadata aggregation and cataloguing. This page describes AWF, the Web 2.0 framework for simplifying the development of metadata-oriented web applications.

Getting started

Requierments

Make sure that Java and Ant are installed:

java -version
ant -version

Installing

  1. Setup AWF:

mkdir myapp
cd myapp
curl https://rawgit.com/ami-team/AMIWebFramework/master/tools/build.xml > build.xml
ant setup-prod    # setup AWF in prod. mode
# or:
ant setup-debug   # setup AWF in debug mode

  2. Create an index.html page:

ant make-index
You will have to specify the application's title and the AMI endpoint URL (e.g. https://localhost:8443/AMI/FrontEnd).

General principles

AWF can be seen as a sub-application container. Each sub-application is implemented in the directory subapp. They follow the Model–View–Controller (MVC) design pattern dividing them into three interconnected parts in order to separate internal representations of information from the ways that information is displayed:

  • The model represents the data (aka. an AMI command result), and does nothing else.
  • The view displays the model data, and sends user actions to the controller.
  • The controller provides data to the view, and interprets user actions.

AMI Commands

subapp/<mySubapp>/css

subapp/<mySubapp>/twig

subapp/<mySubapp>/js

The model, and possibly the view, can be reused without modification.
